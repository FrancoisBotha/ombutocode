# Contributing to Ombuto Code

Thanks for your interest in contributing to Ombuto Code! This project is maintained by Francois Botha and released under the Apache License 2.0.

## Before you contribute

Before we can accept your contribution, you'll need to sign the [Contributor License Agreement (CLA)](./CLA.md). This is a one-time process per contributor. The CLA confirms that:

- You have the right to submit the code you're contributing.
- You grant Francois Botha a broad license to use, modify, and redistribute your contribution, including the right to relicense it as part of future commercial or derivative works.
- You retain copyright in your contribution.

### How to sign

When you open your first pull request, the [CLA Assistant](https://cla-assistant.io/) bot will post a comment with a link to sign electronically. Signing is a one-click process tied to your GitHub account, and you only need to do it once across all your future contributions to this project.

Your pull request cannot be merged until the CLA is signed.

## How to contribute

1. **Fork the repository** and create a feature branch from `main`.
2. **Make your changes**, keeping commits focused and well-described.
3. **Test your changes** locally before opening a pull request.
4. **Open a pull request** against `main` with a clear description of what the change does and why.
5. **Sign the CLA** when prompted by the bot.
6. **Respond to review feedback** — most PRs will go through at least one round of review.

## Coding standards

- Follow the existing code style in the file you're editing.
- Keep changes focused. If you spot unrelated issues, open a separate issue or PR for them.
- Add or update tests where applicable.
- Update documentation if your change affects user-facing behaviour.

## Reporting bugs and requesting features

Please use GitHub Issues for bug reports and feature requests. When reporting a bug, include:

- What you were trying to do
- What you expected to happen
- What actually happened
- Steps to reproduce
- Your environment (OS, version, etc.)

## Code of conduct

Be respectful, constructive, and patient. Disagreement is fine; personal attacks are not. Maintainers reserve the right to remove comments, commits, or contributors that violate this principle.

## Questions

If you have questions about contributing that aren't covered here, open a GitHub Discussion or reach out to Francois Botha directly.

Thanks again for contributing to Ombuto Code.
